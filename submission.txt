What to write in the IN-SPACe application
The application asks you to demonstrate how your solution leverages AI/ML to solve critical challenges. Structure your submission around these five points directly from the document's evaluation criteria:
a) Socio-economic benefit and community need — India has 50+ satellites operational. ISRO currently uses manual conjunction screening. Automating this reduces mission risk and extends satellite life.
b) Indigenous technology — trained model weights are yours. Risk scoring logic is yours. CDM feature engineering is yours. The plan to replace LightGBM inference with an ONNX C runtime removes the foreign dependency.
c) Significant quantifiable improvement — your model achieves >90% alert accuracy vs the broken 16.2% baseline. State this explicitly. Also: current manual ISRO screening takes hours; your system runs in under 100ms per conjunction.
d) Scale-up potential — the same model can protect any ISRO satellite with a configuration change. The ground training pipeline re-trains on new CDM data automatically.
e) Feasibility using space technology — your simulator/orbital.py uses SGP4 propagation, which is the same algorithm ISRO uses. Your CDM features map directly to CCSDS standard. The edge deployment on Raspberry Pi demonstrates hardware feasibility.
